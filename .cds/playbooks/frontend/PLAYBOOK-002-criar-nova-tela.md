# PLAYBOOK-002 - Criar Nova Tela

## Objetivo
Orientar a criação de uma nova tela seguindo o padrão visual e estrutural da plataforma CDS.

## Quando utilizar
Sempre que for adicionar uma nova tela ao frontend.

## Pré-requisitos
- Design System disponível.
- Estrutura do módulo conhecida.

## Entradas
- Requisitos visuais e funcionais.
- Contexto do módulo.

## Saídas Esperadas
- Tela inicial implementada com padrão visual.
- Documentação básica do componente.

## Passo a Passo
1. Definir contexto e objetivo da tela.
2. Reaproveitar componentes do Design System.
3. Estruturar a tela seguindo o padrão do módulo.
4. Conectar a tela à API ou camada de serviço.
5. Validar estados, acesso e testes básicos.

## Checklist
- [ ] Componente reutilizável identificado.
- [ ] Tela estruturada.
- [ ] Integração básica validada.

## Critérios de Aceite
- A tela segue o padrão do Design System.
- Os estados principais foram avaliados.

## Testes Obrigatórios
- Teste de renderização.
- Teste de interação básica.

## Documentação
- README do módulo.
- Referência ao componente.

## ADRs Relacionados
- ADR-005

## Skills Relacionadas

## RFCs Relacionadas

## Anti-padrões
- Criar componentes sem padronização.
- Implementar lógica diretamente na tela.

## Observações
Este playbook cobre a execução estrutural da tela, não a criação de regra de negócio.

- Expandir para telas com fluxo mais complexo.
