# Sprint S-6.2 — LIP (Localizador Inteligente de Produtos)

## Objetivo

Substituir o fluxo manual **Digitar → Buscar → Adicionar** por um componente corporativo reutilizável com busca em tempo real, lista rica de resultados e navegação por teclado.

**Escopo:** apenas experiência do usuário — sem alteração de regras de negócio.

---

## Componente corporativo

```
frontend/shared/components/LIP/
├── index.js              # Componente principal (UI + teclado + qty bar)
├── LipSearchService.js   # API, debounce, cache 60s, paginação
├── lipMappers.js         # highlight, agrupamento, normalização
└── lipHistory.js         # localStorage — últimos 12 produtos
```

### Uso

```javascript
const LIP = require('../../../../shared/components/LIP');

const lip = LIP.create({
  onSelect: (produto, quantidade) => { /* adicionar item */ }
});
lip.mount(document.getElementById('lip-host'));
```

### API pública

| Método | Descrição |
|--------|-----------|
| `LIP.create(options)` | Cria instância |
| `mount(el)` | Monta no DOM |
| `destroy()` | Remove listeners e DOM |
| `focus()` | Retorna foco ao campo de busca |

### Opções

| Opção | Padrão | Descrição |
|-------|--------|-----------|
| `onSelect(produto, qty)` | — | Callback ao confirmar produto |
| `minChars` | `2` | Mínimo de caracteres para buscar |
| `debounceMs` | `300` | Debounce da digitação |
| `pageSize` | `20` | Itens por página |
| `showHistory` | `true` | Exibe últimos utilizados |
| `showFrequentes` | `true` | Exibe ranking ao focar vazio |

## Backend

### `GET /api/produtos/search`

| Parâmetro | Descrição |
|-----------|-----------|
| `q` | Termo unificado (nome, código, barras, referência, categoria, marca) |
| `offset` | Paginação |
| `limite` | Máximo de resultados (padrão 20) |
| `frequentes=1` | Produtos mais vendidos (30 dias) quando `q` vazio |

Resposta:

```json
{
  "items": [{ "id", "nome", "codigo", "categoria", "marca", "estoque", "preco_venda" }],
  "total": 42,
  "offset": 0,
  "limite": 20
}
```

## Homologação manual

1. Reiniciar servidor (`npm start`) para carregar `/produtos/search`
2. Nova Consignação → step Itens
3. Digitar 2+ caracteres → lista abre automaticamente
4. `↓` `Enter` → barra de quantidade → `Enter` → produto na tabela
5. Foco volta ao campo de busca
6. Clicar em item do histórico → adiciona em 1 clique
7. Campo vazio + foco → produtos frequentes (se houver vendas)
