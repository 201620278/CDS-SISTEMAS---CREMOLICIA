# Sprint S-6.4 — Auditoria Contratos Central de Prestação

## Problema

HTTP 400 em `POST /consignacoes/:id/prestacao/abrir` e `.../pagamento` sem mensagem clara no frontend.

## Causas raiz identificadas

| # | Causa | Correção |
|---|-------|----------|
| 1 | `ConsignacaoResponse` não expunha `prestacaoContasAtiva` | Campo incluído no DTO de resposta |
| 2 | Frontend chamava `abrirPrestacao` em todo load sem saber se já estava aberta | Verifica `prestacaoContasAtiva.status === 'ABERTA'` antes de chamar |
| 3 | `AbrirPrestacao` retornava erro `PRESTACAO_JA_ABERTA` em reabertura | Abertura idempotente — retorna sucesso se já aberta |
| 4 | `RegistrarPagamentoRequest` exigia `typeof valor === 'number'` | Coerção de string numérica (`"150,50"` → `150.5`) |
| 5 | Controller usava `HttpResponse` sem import | Substituído por `StandardResponse` + `responderValidacao()` |
| 6 | Erros de domínio sempre HTTP 400 | `StandardResponse.domainError` usa `mapToStatusCode` |
| 7 | `extractErrorMessage` genérico | Inclui `code` e `field` na mensagem |

## Contratos padronizados

### POST `/prestacao/abrir`

**Body:**
```json
{ "usuarioId": "1", "motivo": null }
```

**Resposta sucesso:** `{ success: true, data: { consignacao, grupoPrestacaoContas, idempotente? } }`

### POST `/prestacao/pagamento`

**Body:**
```json
{
  "valor": 150.50,
  "formaPagamento": "DINHEIRO",
  "observacao": null,
  "usuarioId": "1"
}
```

`valor` aceita número ou string (`"150,50"`).

**Erro validação:**
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Campo 'valor' é obrigatório.",
    "details": { "fields": [{ "field": "valor", "message": "..." }] }
  }
}
```

## Regra de negócio (inalterada)

Pagamento exige saldo > 0 (vendas registradas). Mensagem melhorada:
*"Não há saldo em aberto... Registre vendas na prestação antes de registrar o pagamento."*
